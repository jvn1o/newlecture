* 수업 주요 내용

​
- 자바 프로그래밍: 컴퓨터 프로그램을 만드는 행위


- 연산자: 컴퓨터와 계산기의 차이, 즉 일괄적인 계산이 가능한가의 차이 

컴퓨터는 일괄 처리가 가능하다, RAM이 존재.


- 객체지향 프로그래밍: 코드 구조화로 세분화하였을때, 각 코드에 책임감을 부여하여 정리하는 것.

(시각화하여 그림으로 표현하는 것이 직관적이므로 OOA/D & UML 등을 사용하여 나타낸다)

​
- 코드 구조화: 캡슐화(은닉화 & 고립화)가 필수적, 다른 코드의 기능을 사용할 수 있으나 구조를 파악할 수는 없다. 

반대로 서로에게 영향을 주지 않는다. (유지보수에는 용이해지고, 오류 대처가 손쉬워진다)

​
- 자바 번역기 & 개발 도구 무료 사이트: openjdk.org

adoptium.net - JDK.msi (실전용) / JDK.zip (학습용)

​
- 환경 변수(시스템 - 고급 시스템 설정 - 환경 변수): Java를 사용한다면 JAVA_HOME이라는 약속된 위치에 환경 변수를 저장해야 한다.

환경 변수는 외부 명령어를 설정해 줄 수 있는데, 이를 이용하여 해당 프로그램의 폴더 위치가 아니어도 프로그램을 실행할 수 있도록 설정할 수 있다.


- class 파일(결과) / java 파일(실행)

​
- 한글때문에 java 파일이 에러나온다면, 해당 파일의 인코딩을 UTF-8로 바꾸어 실행해보면 된다.

​

* 1주차 조별 스터디 과제 (개인 풀이)

​
자바 프로그래밍

-> 자바를 이용하여 컴퓨터 프로그램을 만드는 행위

​


프로그래밍이란?

-> 후에 쉬운 처리를 위하여, 사전에 원하는 절차를 작성해놓는 행위

​
​

컴퓨터란?

-> 메모리 저장이 가능한 연산 장치

​


컴퓨터 프로그래밍이란?

-> 메모리 저장이 가능한 컴퓨터라는 연산 장치로, 사전에 원하는 절차를 작성해놓는 행위

​
​

자바를 이용한다는 의미는?

-> 사람이 컴퓨터 프로그래밍을 수월하게 해줄 수 있는 소프트웨어 도구를 이용한다는 것

​
​

JDK란?

-> 자바 개발 키트의 줄임말, 자바 컴파일러(javac)를 사용하기 위한 소프트웨어 도구

​
​

컴파일러란?

-> 프로그래밍 언어, 즉 사람이 이해하기 쉬운 고급 언어(프로그래밍 언어)로 작성된 소스 코드를 컴퓨터가 이해하기 쉬운 저급 언어(기계어)로 변환하는 툴이다

​
​

컴파일러는 몇 세대 언어?

-> 3세대 언어

​
​

물리언어란?

-> 기계와 직접적으로 대응되는 언어, 하드웨어에 대한 지식이 요구된다


​

논리언어란?

-> 기계와 독립적인 언어로, 하드웨어에 대한 지식이 필수적이지 않다

​


기계어란?

-> 1세대 언어이며 컴퓨터가 이해하기 쉬운 저급 언어, 이진수로 표현한다


​

어셈블리어란?

-> 2세대 언어이며 기계어가 인간에게 가독성이 떨어지기에, 기계어와 1:1 대응하여 인간에게 편의를 제공하기 위해 존재한다

이때 번역기로는 어셈블러가 사용된다


​

3세대 언어란?

-> 절차적 언어, 고급 언어로써 인간을 위해 만들어진 언어이다. 특정한 기계에 구애받지 않고 프로그래밍을 할 수 있고, 코드의 재사용 & 유지보수가 용이해졌다. 반대로 문법 구조로 이루어져있기 때문에 컴파일러와 같은 번역기가 필요하다는 단점이 있다.

​


고급언어란?

-> 인간이 이해하기 쉽도록 만들어진 언어, 컴퓨터가 이해하기 쉬운 저급언어와 반대되는 개념이다.


​

고급언어를 사용하여 컴퓨터 프로그램을 만들 때의 장점은?

-> 인간을 위한 언어이므로, 가독성이 좋고 유지 보수에 용이하다 


​

기계어를 사용하여 컴퓨터 프로그램을 만들 때의 장점은?

-> 컴퓨터를 위한 언어이므로, 따로 번역할 필요없이 처리할 수 있어 고급언어를 사용했을때보다 상대적으로 속도가 빠르다


​

시스템 환경변수란?

-> 시스템을 사용하는 모든 이용자에게 적용되어 참조되는 동적 변수이다


​

path 환경변수의 역할은?

-> path 환경변수 내의 환경변수를 우선순위에 따라 차례대로 읽도록 한다


​

디렉토리를 이동할 때 사용하는 도스 명령어는?

-> cd (change directory)

​
​

디렉토리 목록을 나열하는 도스 명령어는?

-> dir (directory)
